# Warinporn Kanoksaksopon — Executive Insight Hub

A premium, static personal-brand website **and** an editable, AI-assisted
article system — built to run for free on **GitHub Pages** (no paid backend).

---

## 2. How the system works

Everything reads from **one file**: `assets/articles.json`.
Each article supports: `title`, `slug`, `metaTitle`, `metaDescription`,
`category`, `date`, `readingTime`, `cover`, `tags`, `excerpt`, `body`, and
`draft` (true/false).

- The **homepage** and **Insight Hub** read that file and render the latest
  articles automatically (client-side — works the instant you push).
- The **build script** turns each non-draft article into a real, static
  `/insights/<slug>/` page with its own `<title>`, meta description,
  Open Graph tags, and Article JSON-LD — which is what makes you visible to
  **Google and AI search**.

**6 categories** (already wired into the filter and the CMS dropdown):
Battery Industry Insight · Automotive Aftermarket · Marketing Strategy ·
Dealer Development · Business Growth · Leadership.

## 3. The three options

### Option A — Static (works immediately, zero setup)
Just push the `site/` folder to a GitHub repo and enable Pages. The homepage
and hub render from `articles.json` right away. The 5 sample article pages are
already generated in `/insights/`.

### Option B — GitHub Actions automation (recommended)
Two workflows are included:
- **`build.yml`** — on every push, regenerates all article pages + sitemap and
  deploys. So you only ever edit `articles.json`; the SEO pages build
  themselves.
- **`daily-draft.yml`** — once a day, creates a new **draft** article and opens
  a Pull Request for you to review. Add a repo secret `ANTHROPIC_API_KEY` to get
  a full AI-written draft; without it you still get a structured starting stub.

### Option C — Decap CMS (visual editing, no code)
Open `https://YOUR-SITE/admin/` to add and edit articles in a friendly form
(with a Draft toggle and image upload). Edit `admin/config.yml` and set your
`repo: YOUR-GITHUB-USERNAME/YOUR-REPO`, then connect GitHub auth (see the
comments in that file).

## 4. Publish an article — step by step

**The simple way (edit the data file):**
1. Open `assets/articles.json`.
2. Copy an existing article block inside `"articles": [ ... ]`.
3. Change `title`, `slug` (lowercase-with-dashes), `category`, `date`,
   `excerpt`, `metaDescription`, and `body`. Keep `"draft": false` to publish.
4. **If using Option B:** commit/push — the site rebuilds automatically. Done.
   **If building locally:** run `node tools/build.js`, then commit everything.

**The visual way (Option C):** go to `/admin/`, click **New**, fill in the
form, switch **Draft** off, and **Publish**. The build workflow does the rest.

**Editing before publishing:** set `"draft": true` while you work — it stays
off the live site — then flip it to `false` when ready.

**Run a local preview:**
```bash
cd site
node tools/build.js
python3 -m http.server 8000      # then open http://localhost:8000
```
(A local server is needed because the pages fetch `articles.json`.
`file://` won't work, but GitHub Pages will.)

## 7. "Request Executive Resume" form

Instead of an open PDF download, visitors request your profile through a
qualifying form (Full Name, Company, Position, Business Email, Purpose). This
filters for serious recruiters, search firms, and partners.

**Out of the box** it opens a pre-filled email to `peemaii123@gmail.com` so no
request is lost — zero setup required.

**Recommended (seamless capture, still free):** connect a form service so
submissions arrive without the visitor's email client.
1. Get a free access key at **web3forms.com** (instant, no account needed) — or
   use **formspree.io**.
2. Open `assets/main.js`, find the `Request Executive Resume` block, and set:
   ```js
   var RR_ENDPOINT   = 'https://api.web3forms.com/submit';
   var RR_ACCESS_KEY = 'your-web3forms-key';
   ```
3. Submissions now land in your inbox, and the visitor still sees the
   thank-you message. You reply with the resume personally.

The confirmation shown after submitting:
*"Thank you for your interest. I will review your request and share my
professional profile shortly."*
